FAQ

Frequently Asked Questions

Have questions about licensing, deployment models, or support tiers? Find answers below.

What is WASL's licensing model?

WASL operates on an open-core model. Teams can start with the open foundation, then license enterprise capabilities for collaboration, tenant governance, hybrid operations, advanced AI controls, and support.

Can I deploy WASL in our own private cloud?

Yes. Enterprise deployments can run across private cloud, VPCs, on-premises environments, and edge locations while keeping governance, release control, and observability centralized.

What is the difference between a direct proxy and a flow?

Direct proxy is ideal for secure pass-through APIs. Integration flows are used when an API needs transformation, enrichment, orchestration, or connector-backed backend work.

How does the AI Gateway semantic cache work?

Available in our Enterprise tier, the semantic cache uses embedding models to parse incoming natural language prompts. If a semantically similar prompt was recently evaluated, the gateway serves the cached result directly, saving provider costs and reducing latency.

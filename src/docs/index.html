WASL Documentation

WASL is an open-source API gateway with integration runtime and AI gateway capabilities. These docs are written for end users, not internal engineering teams.

Start here

Use the quickstart to create your first API route, AI route, and deployment flow.

Product model

WASL combines API Gateway, AI Gateway, and integration runtime in one control plane and one operating model.

What WASL is

WASL gives platform teams one place to manage API traffic, integration logic, and AI traffic. Instead of running a gateway product, a separate integration runtime, and an AI control layer, you operate one platform with shared governance, deployment, and observability.

Core surfaces

  • API Gateway for routing, policy enforcement, and release control.
  • AI Gateway for LLM routing, guardrails, quotas, streaming, caching, and prompt governance.
  • Flow Runtime for JavaScript-based integration logic, targets, adapters, and event-driven execution.
Recommended path

If you are new to the platform, go to Quickstart first. It is the fastest way to understand the product model.

How the product is organized

End users mainly work through the Studio control plane. Studio is where you create bundles, APIs, flows, policies, targets, adapters, AI routes, and deployments. The gateway runtime then serves traffic based on those deployed assets.

Typical workflow

  1. Create an API route or AI route in Studio.
  2. Attach policies and configure runtime behavior.
  3. Test in Studio.
  4. Deploy to an environment such as dev or staging.
  5. Invoke the route through the gateway hostname for the selected tenant and environment.

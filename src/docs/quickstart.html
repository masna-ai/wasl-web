Quickstart

Get started with WASL

This quickstart shows the shortest path to create a route, test it, and deploy it through the gateway.

1. Sign in to Studio

Open Studio, sign in, select your tenant, and choose the environment you want to work in. For most teams that starts with dev.

2. Create your first API route

Create a new bundle, add an API endpoint, and let WASL generate the initial flow.

GET /orders
Bundle: order-platform
Flow: ListOrders

3. Add simple flow logic

Edit the generated flow and return a simple JSON payload first. Once the route works, you can add targets or adapters.

response.status = 200;
response.setHeader('content-type', 'application/json');
response.body = {
  ok: true,
  source: 'wasl'
};

4. Test in Studio

Use the built-in test experience from the API or flow detail view. This validates route logic before deployment.

5. Deploy to an environment

Deploy the current revision to an environment such as dev. WASL tracks deployed revisions per environment.

6. Invoke through the gateway

Routes are invoked through the gateway hostname using tenant and environment context.

curl --request GET \
  --url http://acme-corp.dev.localhost:8080/orders

Production-style consumer access (recommended)

  1. Create a consumer product and include operations for your deployed bundle.
  2. Set product default limits (for example: 1000 requests every 60 seconds).
  3. Create a consumer app and subscribe it to that product.
  4. Issue app credential (API key or OAuth client) and test through the portal or direct gateway call.

Rate and quota limits are enforced at runtime from the subscribed plan; limit breaches return 429.

Use docs/testing/CONSUMER_ACCESS_E2E_GUIDE.md as the single end-to-end validation guide.

Optional: expose it through the Developer Portal

After you publish a bundle, you can expose it through one of three portal models:

  • http://localhost:3000/ for shared portal mode
  • http://localhost:3000/?tenant=<tenantId-or-slug> for tenant-scoped mode
  • http://localhost:3000/?portal=<portalId-or-slug> for a managed customer portal
Next step

Continue with API Gateway, AI Gateway, or Developer Portal depending on what you are building.

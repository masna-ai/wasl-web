Quickstart

Get started with WASL

This quickstart shows the shortest path to create a route, test it, and deploy it through the gateway.

1. Sign in to Studio

Open Studio, sign in, select your tenant, and choose the environment you want to work in. For most teams that starts with dev.

2. Create your first API route

Create a new bundle, add an API endpoint, and let WASL generate the initial flow.

GET /orders
Bundle: order-platform
Flow: ListOrders

3. Add simple flow logic

Edit the generated flow and return a simple JSON payload first. Once the route works, you can add targets or adapters.

response.status = 200;
response.setHeader('content-type', 'application/json');
response.body = {
  ok: true,
  source: 'wasl'
};

4. Test in Studio

Use the built-in test experience from the API or flow detail view. This validates route logic before deployment.

5. Deploy to an environment

Deploy the current revision to an environment such as dev. WASL tracks deployed revisions per environment.

6. Invoke through the gateway

Routes are invoked through the gateway hostname using tenant and environment context.

curl --request GET \
  --url http://acme-corp.dev.localhost:8080/orders
Next step

Continue with API Gateway or AI Gateway depending on what you are building.

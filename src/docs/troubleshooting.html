Troubleshooting

Diagnose common runtime and configuration issues

Use this section when a route works in Studio but not in gateway, a policy behaves differently than expected, or environment-specific runtime settings are missing.

Common checks

  • Confirm the correct tenant and environment are selected.
  • Confirm the revision is deployed to the target environment.
  • Verify targets, certificates, and adapter configs for that environment.
  • Check tracing and analytics for the route correlation ID.

API Gateway issues

  • If the wrong route behavior is returned, confirm the active deployment revision for that environment.
  • If a target server is missing, verify the target alias exists in the selected environment.
  • If a policy behaves differently than expected, check whether the deployed revision actually includes the latest policy change.

AI Gateway issues

  • If a route returns provider errors, verify the configured base URL, API key source, and selected provider model.
  • If streaming behavior is unexpected, confirm the request includes the intended stream and streamMode values.
  • If auth blocks the route, check the attached auth config and the actual request headers reaching the gateway.

Integration and adapter issues

  • For JDBC, verify driver path, credentials, port, and that the URL starts with jdbc:.
  • For Kafka, confirm broker endpoints, network path, and whether the listener belongs to a deployed revision.
  • If Studio test mode works but runtime does not, compare the tenant and environment context used in both executions.

Best debugging path

  1. Identify the tenant, bundle or route, and environment.
  2. Confirm deployment state and revision.
  3. Replay the request with curl using the runtime hostname.
  4. Use tracing to inspect policy execution, flow execution, and adapter calls.
  5. Fix the environment-specific config before changing route logic if the failure is infrastructure-related.

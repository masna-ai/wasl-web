Data plane engine

Apigee-class management. Lightweight runtime.

WASL is a high-performance, multi-tenant API Gateway combined with an embedded FlowJS iPaaS runtime. It runs on lightweight Jetty 12 and Project Loom virtual threads to serve millions of requests without the overhead of heavy enterprise middleware.

Execution Mode Spectrum

Three paths. Zero compromise.

Most gateways assume every route requires script execution. WASL optimizes resources by letting you pick the cheapest execution mode per endpoint, handled natively by the router.

01

Direct Proxy

Pass-through to a backend target with policies. Cheap, high throughput, zero script overhead.

02

Flow Orchestration

Full GraalJS JavaScript scripting for custom orchestration, transformations, and connector plugins.

03

Mock Sandbox

Serve static mock payloads for prototyping, sandbox testing, and contract-first development.

flows/orchestrate-payment.js javascript 
// Invoked in Flow v3 GraalJS engine
const order = context.getVariable('request.content');

// Call external ERP service using environment-aware target resolution
const erpResponse = http.postJson('erp-service-target', {
  path: `/orders/${order.id}/sync`,
  body: { orderId: order.id, amount: order.total }
});

if (erpResponse.status === 200) {
  // Invoke built-in Kafka integration plugin to publish event
  adapters.invoke('kafka', {
    operation: 'produce',
    topic: 'orders.synced',
    value: JSON.stringify({ orderId: order.id, erpId: erpResponse.data.id })
  });
  
  context.setVariable('response.content', { success: true, erpSynced: true });
} else {
  context.setVariable('response.status', 500);
  context.setVariable('response.content', { error: 'ERP Sync Failed' });
}

Target Resolution

Three-Level Target Architecture

Decouple logical service references from physical endpoints. In WASL, your flows and routes reference target aliases, and the platform maps them dynamically across environments:

Flow Route: References logical service name (e.g., 'erp-service') Bundle Reference: Binds the service name to a bundle target configuration Environment Mapping: Maps the bundle target configuration to physical hostnames (e.g., dev vs prod endpoints)
Level 1: Orchestration Code Logical Target Alias http.getJson('customer-db', ...)
Level 2: Bundle Config Target Configuration Timeout rules, retry policies, HTTP settings
Level 3: Environment Overlays Target Server Bindings Dev: internal-dev.db:8080 | Prod: aws-rds.internal

Security Engine

24 Production-Grade Policies

WASL runs policies on every request stage: request inbound, target outbound, target response, client outbound. Reusable Policy Sets let you define templates for standard security.

Authentication & Identity API Keys, Basic Auth, JWT validations, full OAuth2 Server flows, mTLS CA certificate anchors.
Rate Limiting & Quotas Token Bucket, Fixed Window, Sliding Window, and tenant-scoped daily/weekly Quotas.
Validation & WAF JSON Schema compliance, OpenAPI spec matching, IP CIDR filtering, Bot detection, Data Masking.

Model Context Protocol

Govern what agents are allowed to do.

WASL MCP Gateway (`McpGatewayApp`) is an enterprise-grade control point for agent tool access. Convert internal APIs into governed tools, proxy third-party MCP servers with strict SSRF defense, and apply granular grants per agent identity.

Request Platform Access Read MCP Docs

Capabilities

Dual Egress & Inbound Governance

Secure both sides of the Model Context Protocol. Expose internal functions safely or govern outgoing tool execution routes under one roof.

Expose & Serve Tools (Inbound)

Expose curated sets of WASL APIs, FlowJS scripts, or Worker tasks as named tools for LLMs. Instead of leaking raw API endpoints, present high-level, governed tools designed specifically for model context.

Prompt Curations: Write model-friendly tool descriptions and names GraalJS flows: Bind tool execution to custom javascript orchestrators Dynamic schema: Automatically derive parameter schemas from inputs Policy checks: Subject every tool execution to rate limits and auth

Proxy External MCP Servers (Egress)

Enforce a central registry for external MCP connections (GitHub, database connectors, SaaS agents). Block direct, shadow-AI connections and force tool traffic through WASL.

SSRF Protection: Enforce `ProviderEgressGuard` boundaries for remote calls Drift Detection: Diff remote schemas to flag and re-approve altered tools Credential Injection: Inject client secrets only toward validated upstreams Audit Logs: Maintain trace correlation ids across all forwarded steps

Wire Format

JSON-RPC 2.0 over Streamable HTTP

The MCP Gateway natively processes session handshakes, tool listing queries, and tool call executions over standard HTTP paths.

POST /mcp/customer_support_ops (initialize) json-rpc 
// Client initializes session with gateway
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {
    "protocolVersion": "2025-06-18",
    "capabilities": {},
    "clientInfo": { "name": "claude-desktop", "version": "1.0.0" }
  }
}

// Gateway responds with version mapping & session header
// headers: {"Mcp-Session-Id": "sess_80a2b8109f", "Content-Type": "application/json"}
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "protocolVersion": "2025-06-18",
    "capabilities": {
      "tools": { "listChanged": true }
    },
    "serverInfo": { "name": "wasl-mcp-gateway", "version": "0.1.0" }
  }
}
POST /mcp/customer_support_ops (tools/call) json-rpc 
// Agent invokes served tool
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "lookup_customer",
    "arguments": { "id": "cust_123" }
  }
}

// Response from WASL (executes flow and returns result)
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "Customer ID cust_123 resolved: Jane Doe, Status: VIP, Tier: Enterprise."
      }
    ]
  }
}

Spine

The Agent Governance Spine

Traditional API keys grant broad scopes. WASL reorients access around **Agent Identity** and **Tool-Level Grants**:

Agent Registration: Track client identities dynamically in `mcp_agents` Entitlement Grants: Bind allowed/denied/approval-required tools to specific agents Discovery-Time Filtering: `tools/list` returns only tools the requesting agent is authorized to call

Real-Time Data Streams

Expose Kafka streams. Run async jobs.

WASL Event Gateway (`EventGatewayApp`) bridges HTTP clients with backend Kafka clusters. Publish events over simple REST and consume them via Server-Sent Events (SSE). Woven together with the `WorkerApp` scheduling engine to govern background loops.

Runtimes

Dual Engine for Async Architectures

Govern both stream communication and background cron schedules under the same multi-tenant control plane.

Event Gateway (`EventGatewayApp`)

Expose real-time message brokers (Kafka, Confluent, MSK, Redpanda) to lightweight web clients. The gateway handles group coordination, token bucket throttling, and schema validation.

REST-to-SSE: Clients produce with HTTP POST, consume over stream SSE Data Contracts: Enforce JSON Schema, Avro, or Protobuf formats at the edge DLQ Routing: Invalid records are automatically routed to Dead Letter Queues AsyncAPI: Self-documenting channels generate AsyncAPI 2.x documents

Worker Runtime (`WorkerApp`)

A dedicated process designed for out-of-band asynchronous jobs, connector scheduling, and long-running flow orchestrations.

Scheduled Jobs: Cron trigger engines coordinate recurring tasks across clusters Retry & Leases: Job lease locking with automated retry and backoff curves Audited Progress: Track job status and log steps directly inside WASL Studio Kafka triggers: Execute JS flows dynamically on topic message ingestion

Simplified Event APIs

Expose real-time capabilities to developers without requiring heavy Kafka libraries or custom SASL configuration.

POST /events/orders.created/records json 
// Client submits event payload
curl -X POST https://events.acme.local/events/orders.created/records \
  -H "Authorization: Bearer wsl_token_88921a" \
  -H "Content-Type: application/json" \
  -d {
    "records": [
      {
        "key": "cust_123",
        "value": { "orderId": "ord_908", "total": 150.00 }
      }
    ]
  }

// Response from Event Gateway
{
  "status": "published",
  "topic": "orders.created.v1",
  "partition": 2,
  "offset": 14502
}

# Client subscribes to SSE stream using regular EventSource API
